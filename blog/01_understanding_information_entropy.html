Understanding Information Entropy

  • Home
  • Thoughts

In 1948, the genius Claude Shannon introduced an important concept of information entropy in his paper A Mathematical Theory of Communication, in which the information entropy is described as

\[ H(X) = - \sum_{i} p(x_i) \log p(x_i) \] Where

  • \(X\) is a random variable.
  • \( p(x_i) \) is the probability that the outcome of \(X\) is \( x_i \)

    • Represents how “chaotic” the probability distribution is for the random variable \(X\).

    For example, in a case that there are 2 possible outcomes of \(X\):

  • When the probability for both the two outcome is 0.5 then the information entropy reaches its maximum and it is the most chaotic situation: you don’t know what outcome is going to appear since the probability for both case is the same.
  • When the probability of one outcome grow and the another shrink, the entropy value decrease, and the system is less chaotic now, as it start to incline to one of the outcome.
  • When the probability of one of the outcome becomes 1 and another becomes 0, the entropy reaches the minimum, since you know for sure what the outcome will be and the system is not chaotic at all.

    • So why does this simple magic formula exactly represent how chaotic is the probability distribution? We may consider:

    \[ \log \frac{1}{p} \]

    Where we may take log base 2, if we want to consider using binary bits of 0 and 1to represent the outcomes (as we do in the real world computers).

    \[ \mbox{If } p(x_i) = \frac{1}{4} \mbox{ then } \log \frac{1}{p(x_i)} = 2 \]

    Which means we need 2 bits to represents all the possibilities for this random variable \[ \mbox{If } p(x_i) = \frac{1}{8} \mbox{ then } \log \frac{1}{p(x_i)} = 3 \] Which means we need 3 bits to represents all the possibility for this random variable.

    So how many bit do we actually need for all the possible outcomes? We may calculate expected value by:

    \[ \sum_{i} p(x_i) \log \frac{1}{p(x_i)} = - \sum_{i} p(x_i) \log p(x_i) \]

    It means the expectation of how many bits do we need to represent all the outcomes, and it gives us the information entropy H(X).

    We may interpret in this way: the more bit we need to represent all the outcomes, the more chaotic is a system.
